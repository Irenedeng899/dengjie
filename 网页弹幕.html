GitHub Pages 部署教程

  1. 创建GitHub账号（如果没有）
  2. 创建一个新的代码仓库（Repository）
  3. 将您的HTML文件上传到仓库
  4. 进入仓库设置，找到"Pages"选项
  5. 选择主分支（main/master）作为源
  6. 保存设置，等待几分钟后您的网站就会发布

您的网站地址将是：https://您的用户名.github.io/仓库名

注意： GitHub Pages 适合静态网站，但不支持服务器端技术如PHP或数据库。

Netlify 部署教程

  1. 创建Netlify账号（可以使用GitHub账号登录）
  2. 点击"New site from Git"
  3. 连接您的GitHub仓库
  4. 配置构建设置（对于纯HTML项目，无需特殊配置）
  5. 点击"Deploy site"
  6. Netlify会自动分配一个域名，您也可以添加自定义域名

Vercel 部署教程

  1. 创建Vercel账号（可以使用GitHub账号登录）
  2. 点击"Import Project"
  3. 连接您的GitHub仓库
  4. 点击"Deploy"（对于HTML项目，无需特殊配置）
  5. Vercel会自动分配一个域名，您也可以添加自定义域名

传统虚拟主机部署教程

  1. 购买虚拟主机和域名
  2. 使用FTP客户端（如FileZilla）连接您的主机
  3. 将HTML文件上传到public_html或www目录
  4. 通过您的域名访问网站

2. 上传HTML文件

无论选择哪种部署方式，都需要上传HTML文件：

弹幕系统/
├── index.html          # 主页面
├── danmu-system.html   # 弹幕系统页面
├── style.css          # 样式文件（可选）
└── script.js          # 脚本文件（可选）

3. 测试您的网站

部署完成后，请测试：

  • 在不同浏览器中打开网站
  • 使用手机访问网站
  • 测试弹幕提交和显示功能
  • 检查二维码是否能够正确扫描

4. 配置跨设备通信

要使弹幕系统在多设备间正常工作，您需要：

  1. 确保所有设备访问的是相同的URL
  2. 对于本地测试，可以使用ngrok或localhost.run等工具将本地服务器暴露到公网
  3. 对于生产环境，确保您的网站有固定的URL
提示： 对于真正的多设备实时通信，建议使用WebSocket或Firebase等实时数据库服务，而不是localStorage。

弹幕系统使用说明

手机端使用

  1. 使用手机扫描二维码
  2. 在表单中输入姓名和弹幕内容
  3. 选择弹幕颜色
  4. 点击"发送弹幕"按钮
  5. 弹幕将显示在大屏幕上

大屏幕设置

  1. 在电脑或大屏幕设备上打开弹幕系统页面
  2. 点击"全屏模式"按钮获得最佳显示效果
  3. 确保页面保持打开状态以接收新弹幕
  4. 点击弹幕可以查看发送者信息

管理功能

  • 清空弹幕： 移除所有当前显示的弹幕
  • 添加示例： 添加示例弹幕用于测试
  • 导出数据： 将弹幕数据导出为JSON文件
